### 2.2 系统提示词 (System Prompt)

```
You are a contract risk analysis expert for small business owners and freelancers in the US.

## Core Rules
- NEVER provide legal advice. Always include disclaimer: "This is not legal advice."
- Explain everything in simple, plain English that a non-lawyer can understand
- Be concise but thorough - focus on practical risks

## Red Flag Categories to Detect

1. **Unlimited Personal Liability**
   - Look for: personal liability beyond contract value
   - Risk: Personal assets (home, savings) at risk

2. **Overly Broad Non-Compete**
   - Look for: >2 years duration, unlimited geographic scope
   - Risk: Career limitations

3. **Hidden Auto-Renewal**
   - Look for: automatic renewal without clear notice requirements
   - Risk: Unintended contract extension

4. **One-Sided Jurisdiction**
   - Look for: exclusive jurisdiction in counterparty's location
   - Risk: Expensive legal defense

5. **Asymmetric Termination Rights**
   - Look for: unequal notice periods or termination conditions
   - Risk: Unfair power imbalance

6. **Overreaching IP Assignment**
   - Look for: claims on work outside contract scope
   - Risk: Loss of personal/side project rights

7. **Forced Arbitration**
   - Look for: mandatory arbitration, waiver of jury trial
   - Risk: Limited legal recourse

8. **One-Sided Indemnification**
   - Look for: unilateral indemnification obligations
   - Risk: Unlimited liability for counterparty's issues

## Output Format
MUST return valid JSON matching this exact schema:
{
  "overallRiskScore": <0-100 integer>,
  "redFlags": [
    {
      "clauseText": "<exact problematic text from contract>",
      "riskLevel": "<critical|warning|standard>",
      "plainEnglishExplanation": "<2-3 sentences explaining the risk in simple terms>",
      "recommendation": "<one actionable sentence on what to do>"
    }
  ],
  "summary": "<2-3 sentence overall assessment>",
  "top3ThingsToKnow": ["<thing1>", "<thing2>", "<thing3>"]
}

## Risk Level Definitions
- critical: Could cause major financial or legal harm
- warning: Unfavorable but common in industry
- standard: Normal business terms

## Scoring Guidelines
- 0-25: Low risk, standard contract
- 26-50: Moderate risk, some concerns
- 51-75: High risk, significant issues
- 76-100: Critical risk, major red flags
```

## 3. 模块优化详细方案

### 3.1 模块优化总览

**遵循"优先继承"原则，扩展现有模块而非创建新目录**

```

│   ├── OCRService.swift              # ✅ 保持不变
│   ├── RuleEngine.swift              # ✅ 保持不变

├── Views/
│   ├── Scan/
│   │   ├── ScanView.swift            # ✅ 保持不变
```

#### 3.2.2 DocumentScannerView 优化

**需求描述**:
- 复用 `OCRService` 进行文字识别
- 消除内部OCR实现与 `OCRService` 的重复

**生成规则**:
- Coordinator 持有 `OCRService` 实例
- 使用 `async/await` 调用OCR
- 保持现有回调接口不变（向后兼容）

**代码编写示例**:
```swift
// DocumentScannerView.swift 优化后
class Coordinator: NSObject, VNDocumentCameraViewControllerDelegate {
    let onScanComplete: (String) -> Void
    let onCancel: () -> Void
    let ocrService = OCRService()  // 复用现有服务
    
    func documentCameraViewController(_ controller: VNDocumentCameraViewController, 
                                      didFinishWith scan: VNDocumentCameraScan) {
        Task {
            var fullText = ""
            for i in 0..<scan.pageCount {
                let image = scan.imageOfPage(at: i)
                let pageText = await ocrService.extractText(from: image)
                fullText += pageText + "\n"
            }
            await MainActor.run {
                onScanComplete(fullText)
            }
        }
    }
    // ... 其他方法保持不变
}
```

**检验标准**:
- 扫描后文字识别结果与优化前一致
- 代码行数减少30%以上
- 无功能回归

#### 3.2.3 扩展 Constants.swift（继承现有结构）

**需求描述**:
- 在现有 `AppConstants` 中添加API配置
- 添加App Store相关配置
- 保持与现有代码风格一致

**生成规则**:
- 扩展现有 `AppConstants` enum
- 添加嵌套 `CozeConfig` 和 `AppStoreConfig`
- 支持从Info.plist读取敏感配置

**代码编写示例**:
```swift
// Utilities/Constants.swift 扩展后
import SwiftUI

enum AppConstants {
    static let feedbackBaseURL = "https://feedback-board.iocompile67692.workers.dev"
    static let appName = "ContractShield"
    static let privacyPolicyURL = "https://asunnyboy861.github.io/ContractShield-pravicy/"
    static let termsOfUseURL = "https://asunnyboy861.github.io/ContractShield-terms/"
    static let supportURL = "https://asunnyboy861.github.io/ContractShield-support/"

    enum ProductID {
        static let monthly = "com.zzoutuo.ContractShield.pro.monthly"
        static let yearly = "com.zzoutuo.ContractShield.pro.yearly"
        static let all = [monthly, yearly]
    }
    
    enum CozeConfig {
        static var botId: String {
            Bundle.main.object(forInfoDictionaryKey: "COZE_BOT_ID") as? String ?? ""
        }
        static var apiToken: String {
            Bundle.main.object(forInfoDictionaryKey: "COZE_API_TOKEN") as? String ?? ""
        }
        static let baseURL = "https://api.coze.com/v3/chat"
        static var isConfigured: Bool {
            !botId.isEmpty && !apiToken.isEmpty
        }
        static let maxRetries = 3
        static let timeout: TimeInterval = 30
    }
    
    enum AppStoreConfig {
        static var appId: String {
            Bundle.main.object(forInfoDictionaryKey: "APP_STORE_ID") as? String ?? ""
        }
        static var reviewURL: URL? {
            guard !appId.isEmpty else { return nil }
            return URL(string: "https://apps.apple.com/app/id\(appId)?action=write-review")
        }
        static var appVersion: String {
            Bundle.main.infoDictionary?["CFBundleShortVersionString"] as? String ?? "1.0.0"
        }
        static var buildNumber: String {
            Bundle.main.infoDictionary?["CFBundleVersion"] as? String ?? "1"
        }
    }
}

enum FeedbackSubject: String, CaseIterable, Identifiable {
    case bugReport = "Bug Report"
    case featureRequest = "Feature Request"
    case generalFeedback = "General Feedback"
    case question = "Question"
    case other = "Other"

    var id: String { rawValue }
    var displayName: String { rawValue }
}
```

**检验标准**:
- 配置可从Info.plist读取
- 未配置时优雅降级（返回空字符串/nil）
- 与现有代码风格一致

#### 3.2.4 简化分享功能（AnalysisView扩展）

**需求描述**:
- 在 `AnalysisView` 中添加分享按钮
- 支持分享分析摘要文本
- 使用系统分享面板

**生成规则**:
- 使用 SwiftUI `ShareLink` (iOS 16+)
- 生成格式化的文本摘要
- 不创建独立的SharingUtility文件（遵循最小代码原则）

**代码编写示例**:
```swift
// 在 AnalysisView.swift 中添加分享功能
import SwiftUI

struct AnalysisView: View {
    // ... 现有属性 ...
    
    var shareText: String {
        let flags = analysisVM.detectedFlags
        let criticalCount = flags.filter { $0.riskLevel == .critical }.count
        let warningCount = flags.filter { $0.riskLevel == .warning }.count
        
        return """
        ContractShield Analysis Report
        ================================
        Risk Score: \(analysisVM.riskScore)/100
        
        Summary:
        \(analysisVM.summary)
        
        Red Flags Found: \(flags.count)
        - Critical: \(criticalCount)
        - Warning: \(warningCount)
        
        Top Things to Know:
        \(analysisVM.topThingsToKnow.enumerated().map { "\($0.offset + 1). \($0.element)" }.joined(separator: "\n"))
        
        ---
        Generated by ContractShield
        This is not legal advice.
        """
    }
    
    // 在toolbar中添加分享按钮
    ToolbarItem(placement: .topBarTrailing) {
        ShareLink(item: shareText) {
            Image(systemName: "square.and.arrow.up")
        }
    }
}
```

**检验标准**:
- 分享内容格式清晰
- 包含所有关键信息
- 使用系统原生分享面板
